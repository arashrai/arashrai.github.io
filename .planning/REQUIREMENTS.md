# Requirements: Narsh 2026

**Defined:** 2026-05-15
**Core Value:** Guests feel a personal, heartfelt connection to Natalie and Arash's story

## v1 Requirements

### Access & Navigation

- [ ] **ACC-01**: Guest can enter a password on the landing page to access the site
- [ ] **ACC-02**: Day 2 password shows Day 2 event info; full-access password shows all days
- [ ] **ACC-03**: Navigation menu allows moving between site sections
- [ ] **ACC-04**: All pages render well on mobile devices (responsive from day one)
- [ ] **ACC-05**: Warm, playful, colorful visual design foundation and shared styles

### Timeline

- [ ] **TIME-01**: Interactive world map displays pins for each life chapter location
- [ ] **TIME-02**: Scrolling advances the map between stops with smooth fly-to animations
- [ ] **TIME-03**: Each stop shows a clickable photo slideshow (1-3 photos)
- [ ] **TIME-04**: Each stop displays narrative text explaining that phase of life

### Guest Graph

- [ ] **GRAPH-01**: Force-directed network visualization shows connections between guests
- [ ] **GRAPH-02**: Family tree layout mode displays immediate family structure
- [ ] **GRAPH-03**: Filters let guests view subsets by group (family, city, etc.)
- [ ] **GRAPH-04**: Core family member nodes display profile photos and names

### Puzzle Page

- [ ] **PUZZLE-01**: Custom "Clues by Sam" style puzzle as standalone page
- [ ] **PUZZLE-02**: Links to favorite daily puzzles

### Info Pages

- [ ] **INFO-01**: Event schedule page shows times and locations (tier-aware based on password)
- [ ] **INFO-02**: Venue and travel page with directions, hotel recommendations, parking
- [ ] **INFO-03**: Dress code guidance page

## v2 Requirements

(None deferred — all features are in v1 scope, phased across the roadmap)

## Out of Scope

| Feature | Reason |
|---------|--------|

| Email/magic link auth | Replaced by simpler password approach |
| Native mobile app | Web-responsive is sufficient; native app is a different product |
| Video content | Photos only for timeline |
| Real-time features | Static site is sufficient |
