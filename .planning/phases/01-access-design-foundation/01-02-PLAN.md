<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Section page -> auth.js requireAuth | Unauthenticated guests are redirected; but content is in HTML source |
| data-tier="full" -> applyTierVisibility | Day2 guests have full-tier content hidden via JS; content remains in DOM source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | Section page HTML source | accept | All page content is in HTML source. Auth guard only prevents casual browsing, not determined inspection. Accepted for wedding site. |
| T-01-06 | Tampering | data-tier visibility | accept | User could remove display:none or set localStorage tier to "full". Accepted: content tiers are a UX convenience for wedding logistics, not security-sensitive data. |
| T-01-07 | Spoofing | FOUC (brief content flash) | mitigate | body.auth-pending class with visibility:hidden on main prevents content flash before auth.js confirms tier and either redirects or reveals content. |
</threat_model>
