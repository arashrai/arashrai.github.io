## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, fine adjustments |
| sm | 8px | Compact element spacing, form gap |
| md | 16px | Default element spacing, paragraph margin, form margin |
| lg | 24px | Section padding, container horizontal padding |
| xl | 48px | Major layout gaps, hero section spacing |
| 2xl | 64px | Page-level vertical spacing, hero top margin |

Exceptions: Touch targets for the hamburger menu button and password submit button must be minimum 44x44px (WCAG 2.5.5).

**Source:** RESEARCH.md Pattern 3 (CSS Custom Properties), adapted from `--space-*` tokens

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 17px | 400 (regular) | 1.6 | Source Sans 3 |
| Small / Label | 14px | 400 (regular) | 1.4 | Source Sans 3 |
| Heading (h2-h3) | 20px | 700 (bold) | 1.2 | Playfair Display |
| Display (h1) | 28px | 700 (bold) | 1.2 | Playfair Display |

Labels are differentiated from body text by size (14px vs 17px) and context, not by weight. Where additional visual distinction is needed for labels (e.g., form labels, nav metadata), apply `letter-spacing: 0.02em` and `text-transform: uppercase`.

Mobile overrides (below 768px):
- Display (h1): 24px
- Body: 16px (minimum for iOS to prevent auto-zoom on input focus)

Letter spacing on headings: `-0.02em` (carries forward existing pattern from current `narsh2026/index.html`).

Google Fonts load string:
```
https://fonts.googleapis.com/css2?family=Playfair+Display:wght@400;700&family=Source+Sans+3:wght@400&display=swap
```

**Source:** RESEARCH.md Discretion Recommendations (typography), RESEARCH.md Pattern 3 (`--font-*` tokens)

## Color

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-cream` | #FFF8F0 | Page background, gate background, content area |
| Dominant alt | `--color-warm-white` | #FFFDFB | Input field backgrounds |
| Secondary (30%) | `--color-dusty-rose` | #C9928E | Card backgrounds, nav background, hero photo placeholder, coming-soon section |
| Secondary alt | `--color-golden` | #D4A843 | Hover states on secondary surfaces, active nav indicator |
| Accent (10%) | `--color-terracotta` | #C2704F | Primary CTA button, nav active link underline, form submit button |
| Text primary | `--color-text-primary` | #3D2B1F | Headings, body text, input text |
| Text secondary | `--color-text-secondary` | #6B4F3A | Placeholder text, secondary labels, status messages |
| Destructive | N/A | N/A | No destructive actions in this phase |

Accent reserved for:
- Primary CTA button ("Let's go" submit)
- Active navigation link underline/indicator
- Password input focus border ring

Elements that must NOT use accent:
- Body text
- Backgrounds
- Navigation link default state
- Borders (except focus rings)

**Source:** CONTEXT.md D-04 (sunset warm tones), RESEARCH.md Pattern 3 (hex values)

## Shape & Motion

| Property | Token | Value |
|----------|-------|-------|
| Border radius (small) | `--radius-sm` | 6px |
| Border radius (medium) | `--radius-md` | 12px |
| Border radius (large) | `--radius-lg` | 20px |
| Shadow (soft) | `--shadow-soft` | 0 2px 8px rgba(61, 43, 31, 0.08) |

| Transition (fast) | `--transition-fast` | 150ms ease |

Usage rules:
- Input fields: `--radius-sm` (6px)
- Buttons: `--radius-sm` (6px)
- Cards / content containers: `--radius-md` (12px)
- Hero photo placeholder: `--radius-lg` (20px)
- Hover effects on buttons/links: `--transition-fast`
- Nav menu open/close: `--transition-normal`

**Source:** CONTEXT.md D-05 (rounded corners, soft shadows, gentle hover effects), RESEARCH.md Pattern 3

## Layout & Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile | < 768px | Single column, hamburger nav, stacked form, 16px body text |
| Desktop | >= 768px | Centered content (max-width 560px), horizontal nav bar, 17px body text |

Gate page layout (mobile):
- Full-width hero photo placeholder
- Centered heading and tagline below photo
- Full-width password form (input stacked above button if needed, or side-by-side if space allows)
- 24px horizontal padding

Gate page layout (desktop):
- Centered column, max-width 560px
- Hero photo, heading, tagline, form vertically stacked
- Form: input and button side-by-side with 8px gap

Section pages layout (both breakpoints):
- Site header pinned at top (not sticky/fixed -- scrolls with content)
- Main content centered, max-width 560px, 24px horizontal padding
- 64px top margin from header to content

**Source:** RESEARCH.md Architecture Patterns, existing `narsh2026/index.html` layout pattern (max-width 560px, margin 96px auto)

## Interaction States

### Password Form

| State | Visual Treatment |
|-------|-----------------|
| Default | Input: 1px `--color-dusty-rose` border, `--color-warm-white` background. Button: `--color-terracotta` background, white text |
| Input focused | Input border changes to 2px `--color-terracotta`, subtle `--shadow-soft` |
| Button hover | Button background darkens ~10% (use CSS `filter: brightness(0.9)`) |
| Button active (pressed) | Button background darkens ~15% (`filter: brightness(0.85)`) |
| Submitting | Button text changes to "..." or shows subtle opacity reduction, `cursor: not-allowed` |
| Error (wrong password) | Status message appears below form in `--color-text-secondary`, input border remains neutral (not red -- keep it warm and friendly per D-12) |
| Success | Gate div hides, welcome content appears. No animation needed -- instant swap. |

### Navigation

| State | Visual Treatment |
|-------|-----------------|
| Desktop default | Horizontal link row, `--color-text-primary`, no underline |
| Desktop hover | Link color shifts to `--color-terracotta`, `--transition-fast` |
| Desktop active (current page) | 2px `--color-terracotta` underline below link text |
| Mobile closed | Hamburger icon visible, nav links hidden |
| Mobile open | Full-screen overlay with `--color-cream` background, nav links stacked vertically, generous tap targets (48px height per link) |
| Mobile nav link hover/tap | Background color shifts to `--color-dusty-rose` at 20% opacity |

### Auth-Pending Flash Prevention

- All section pages load with `<body class="auth-pending">`
- CSS rule: `body.auth-pending main { visibility: hidden; }`
- `auth.js` removes `auth-pending` class after confirming tier
- Uses `visibility: hidden` (not `display: none`) to prevent layout shift

**Source:** RESEARCH.md Pitfall 2 (FOUC), RESEARCH.md Pattern 4 (Navigation), CONTEXT.md D-05 (gentle hover effects)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Language | `lang="en"` on `<html>` |
| Input autocomplete | `autocomplete="off"` on password field (not a login credential) |
| Required fields | `required` attribute on password input |
| Error announcement | `aria-live="polite"` on status message container |
| Nav toggle state | `aria-expanded="true/false"` on hamburger button |
| Nav toggle label | `aria-label="Menu"` on hamburger button |
| Nav region | `aria-controls="site-nav"` on hamburger button |
| Keyboard nav close | Escape key closes mobile nav overlay |
| Focus management | Focus returns to hamburger button when nav closes |
| Touch targets | Minimum 44x44px for hamburger button and submit button |
| Color contrast | `--color-text-primary` (#3D2B1F) on `--color-cream` (#FFF8F0): ratio ~12:1 (passes AAA) |
| Font size | Minimum 16px on mobile to prevent iOS auto-zoom |

**Source:** RESEARCH.md Pitfall 5 (Hamburger Accessibility), CLAUDE.md accessibility patterns
