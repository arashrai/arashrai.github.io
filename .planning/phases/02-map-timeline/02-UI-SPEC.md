### Phase 2 Journey Line Colors (new tokens)

These are NOT part of the sunset palette. They represent Arash's and Natalie's family wedding-day colors and must visually pop against the warm-toned map.

| Token | Value | Usage |
|-------|-------|-------|
| `--color-journey-arash` | #2A9D8F | Arash's journey line, Arash's stop dots, teal-coded stop name accent |
| `--color-journey-natalie` | #D4A843 | Natalie's journey line, Natalie's stop dots, gold-coded stop name accent |

Color rationale:
- Teal (#2A9D8F): A warm-leaning teal that harmonizes with the sunset palette while remaining distinctly "cool" enough to read as teal. Passes WCAG AA contrast against the warm map background.
- Gold (#D4A843): The same `--color-golden` already in the Phase 1 palette. Reused for Natalie's journey line to maintain palette cohesion. If contrast issues arise on warm map tiles, stroke the line with a 1px darker outline (#B8922F).

### Map Tile Color Direction

The custom map style must use these color ranges to feel warm and harmonize with the site palette:

| Map Element | Color Range | Notes |
|-------------|-------------|-------|
| Land | #F5E6D3 to #EDD9C4 | Warm cream/sand, close to `--color-cream` but distinct |
| Water | #D4BFA8 to #C9B094 | Warm tan, NOT blue -- keeps the entire map in sunset tones |
| Country borders | #E0CDB8 | Subtle, same hue family as land but slightly darker |
| Labels (country names) | #8B7355 | Warm brown, sparse -- only major countries labeled |
| Labels (city names) | Hidden | Do not show city labels -- stop pins serve this purpose |

Accent reserved for:
- Active timeline dot (the currently active stop in the timeline bar)
- Active carousel dot (current photo indicator)
- Content panel "scroll to begin" prompt

Elements that must NOT use accent:
- Journey lines (use journey-specific teal/gold)
- Map pins (use journey-specific teal/gold)
- Narrative body text
- Timeline bar background

**Source:** Phase 1 UI-SPEC color contract, CONTEXT.md D-11 (warm-toned map), D-12 (teal/gold journey colors), D-13 (color-coded dots)

## Shape & Motion

### Inherited (from Phase 1)

| Property | Token | Value |
|----------|-------|-------|
| Border radius (small) | `--radius-sm` | 6px |
| Border radius (medium) | `--radius-md` | 12px |
| Border radius (large) | `--radius-lg` | 20px |
| Shadow (soft) | `--shadow-soft` | 0 2px 8px rgba(61, 43, 31, 0.08) |
| Shadow (medium) | `--shadow-medium` | 0 4px 16px rgba(61, 43, 31, 0.12) |
| Transition (fast) | `--transition-fast` | 150ms ease |
| Transition (normal) | `--transition-normal` | 250ms ease |

### Phase 2 Additions

| Property | Token | Value |
|----------|-------|-------|
| Map fly-to duration | `--map-fly-duration` | 2000ms |
| Map fly-to easing | N/A (Mapbox default) | ease-in-out curve |
| Line draw duration | `--line-draw-duration` | 1200ms |
| Content cross-fade (mobile) | `--content-fade-duration` | 300ms |
| Bow animation duration | `--bow-duration` | 1500ms |

Usage rules:
- Photo carousel container: `--radius-md` (12px)
- Individual photos: `--radius-sm` (6px)
- Content panel: `--radius-md` (12px) on top corners only (panel sits at viewport bottom)
- Timeline bar: no radius (full-width bar at viewport bottom)
- Carousel navigation arrows: `--radius-sm` (6px)

### Animation Sequences

**Scroll-driven stop transition:**
1. User scrolls to next stop trigger point
2. Map begins fly-to animation to new coordinates (2000ms)
3. Journey line draws progressively from previous stop to current (1200ms, starts 400ms into fly-to)
4. Content panel cross-fades to new stop's text and photos (300ms, starts when fly-to reaches 60%)

**Bow animation at convergence (SHAD Valley):**
1. Both teal and gold lines arrive at convergence point
2. Lines cross over each other twice forming a bow shape (1500ms total)
3. A brief scale pulse (1.1x) on the convergence pin dot to draw attention

**Post-convergence intertwining:**
- Lines follow a shared path but oscillate with a sine-wave offset (amplitude ~3px) creating a braided/intertwined appearance
- When lines separate temporarily (different cities), the sine wave straightens and lines diverge to their respective coordinates

**Source:** CONTEXT.md D-02 (progressive line drawing), D-04 (bow animation), D-06 (pinned map)

## Layout & Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile | < 768px | Map and content alternate via cross-fade; timeline bar at bottom; content panel full-width |
| Desktop | >= 768px | Map fills viewport; content panel overlays as a card (right side or bottom-left); timeline bar at bottom |

### Desktop Layout (>= 768px)

```
+-----------------------------------------------+
|  [Site Header: Logo + Nav]                     |
+-----------------------------------------------+
|                                                |
|           WORLD MAP (full viewport)            |
|                            +------------------+|
|                            | Content Panel    ||
|                            | [Photo Carousel] ||
|                            | Stop Name        ||
|                            | Narrative text   ||
|                            +------------------+|
|                                                |
+-----------------------------------------------+
|  [Timeline Bar: dots + year labels]            |
+-----------------------------------------------+
```

- Map: `position: fixed`, fills viewport below header (top: header height, bottom: timeline bar height)
- Content panel: `position: fixed`, bottom-right corner, max-width 420px, max-height 60vh, scrollable if content overflows
- Content panel: `--color-cream` background, `--shadow-medium`, `--radius-md` on top corners
- Timeline bar: `position: fixed`, bottom: 0, full width, height: 48px, `--color-dusty-rose` at 15% opacity background
- Site header: `position: fixed`, top: 0, full width, z-index above map

### Mobile Layout (< 768px)

```
+---------------------------+
| [Site Header]             |
+---------------------------+
|                           |
|   MAP VIEW (full screen)  |
|     or                    |
|   CONTENT VIEW            |
|   [Photo Carousel]        |
|   Stop Name               |
|   Narrative text          |
|                           |
+---------------------------+
| [Timeline Bar]            |
+---------------------------+
```

- Map and content alternate: at each stop, show map fly-to first, then cross-fade to content view (per CONTEXT.md D-07)
- Content view: full viewport (below header, above timeline bar), `--color-cream` background
- Photo carousel: full width minus 32px padding (16px each side)
- Timeline bar: same as desktop, 48px height, fixed bottom
- Cross-fade duration: 300ms (`--content-fade-duration`)

### Photo Carousel Sizing

| Breakpoint | Carousel Width | Carousel Height | Photo Aspect Ratio |
|------------|---------------|-----------------|-------------------|
| Mobile | 100% - 32px padding | Auto (constrained by 3:2 ratio) | 3:2 |
| Desktop | 100% of content panel width - 32px padding | Auto (constrained by 3:2 ratio) | 3:2 |

Photo max-height: 280px on mobile, 240px on desktop (content panel is narrower).

### Scroll Mechanics

- The page uses a tall invisible scroll container to drive the map/content transitions
- Each stop corresponds to ~100vh of scroll distance
- `IntersectionObserver` or scroll position calculation triggers stop transitions
- The map, content panel, and timeline bar are all `position: fixed` -- only the invisible scroll container actually scrolls
- On mobile, touch scroll drives the same mechanism

**Source:** CONTEXT.md D-06 (pinned map), D-07 (responsive panel placement), D-08 (clickable timeline bar)

## Interaction States

### Map

| State | Visual Treatment |
|-------|-----------------|
| Loading | Map area shows `--color-cream` background with centered "Loading the map..." text |
| Ready (initial) | Map centered on a mid-world view showing both origin continents, scroll prompt visible |
| Flying (between stops) | Smooth fly-to animation, 2000ms, map zooms and pans to next stop coordinates |
| At stop | Map centered on stop location, pin dot at 1.3x scale with subtle pulse, journey lines drawn up to this point |
| Error (WebGL) | Map area shows error message on `--color-cream` background, no map rendered |
| Error (tiles) | Map area shows error message, suggests page refresh |

### Content Panel

| State | Visual Treatment |
|-------|-----------------|
| Hidden (pre-scroll) | Not visible; scroll prompt shown instead |
| Entering (desktop) | Fades in with `--transition-normal` (250ms) when first stop is reached |
| Active | Shows current stop's photo carousel + heading + narrative |
| Transitioning | Cross-fade between old and new content (300ms) |
| Mobile map view | Content panel hidden, map visible full-screen |
| Mobile content view | Map hidden, content visible full-screen with cream background |

### Photo Carousel

| State | Visual Treatment |
|-------|-----------------|
| Single photo | No arrows shown, no dots shown (just the photo) |
| Multi-photo default | First photo visible, next arrow visible, prev arrow hidden, dots shown |
| Arrow hover | Arrow background: `--color-text-primary` at 8% opacity, `--transition-fast` |
| Arrow active | Arrow background: `--color-text-primary` at 15% opacity |
| Photo transitioning | Slide animation left/right, 250ms ease |
| Touch swipe | Follows finger position, snaps to nearest photo on release |

### Timeline Bar

| State | Visual Treatment |
|-------|-----------------|
| Default | All dots at 8px diameter, `--color-dusty-rose` at 40% opacity |
| Visited dot | 8px diameter, `--color-dusty-rose` at full opacity |
| Active dot | 12px diameter, `--color-terracotta`, subtle scale transition |
| Dot hover | Tooltip shows stop location name, 14px `--color-text-primary` on `--color-warm-white` background |
| Dot click | Triggers scroll to that stop (animates map + content) |

### Journey Lines

| State | Visual Treatment |
|-------|-----------------|
| Not yet drawn | Invisible (line path exists but stroke-dashoffset hides it) |
| Drawing | Progressive reveal from previous stop to current, 1200ms, eased |
| Drawn (past stops) | Full opacity (1.0), 3px stroke width |
| Active segment | Slightly brighter (opacity 1.0 + subtle glow via line blur layer at 0.3 opacity, 6px width) |

**Source:** CONTEXT.md D-02 (progressive line drawing), D-09 (photo carousel), D-08 (timeline bar)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Language | `lang="en"` on `<html>` (inherited from Phase 1 template) |
| Map aria label | `aria-label="Interactive world map showing Natalie and Arash's journey"` on map container |
| Map role | `role="img"` on map container (map is presentational, not interactive for screen readers) |
| Narrative accessible | All narrative text is in standard DOM elements (not canvas) -- screen readers read it naturally |
| Photo alt text | Every `<img>` has descriptive alt text per the alt text pattern in Copywriting Contract |
| Carousel controls | `aria-label="Previous photo"` / `aria-label="Next photo"` on arrow buttons |
| Carousel state | `aria-live="polite"` on a visually-hidden element announcing "Photo {n} of {total}" on change |
| Timeline bar dots | Each dot is a `<button>` with `aria-label="{Location}, {Year}"` |
| Timeline bar | `role="navigation"` with `aria-label="Story timeline"` on the bar container |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` — skip fly-to animations (instant map jumps), skip line draw animations (instant reveal), disable carousel slide (instant swap) |
| Touch targets | All interactive elements minimum 44x44px (carousel arrows, timeline dots, nav) |
| Keyboard navigation | Tab through timeline dots and carousel arrows; Enter/Space activates; arrow keys cycle carousel photos |
| Color contrast | Journey line colors on warm map: teal #2A9D8F and gold #D4A843 both exceed 3:1 contrast ratio against #F5E6D3 land color |
| Focus visible | All interactive elements show visible focus ring: 2px `--color-terracotta` outline with 2px offset |
| Skip map link | Visually-hidden "Skip to story content" link before the map for keyboard/screen reader users |

**Source:** Phase 1 UI-SPEC accessibility requirements, extended for map-specific concerns

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| N/A | N/A | Not applicable -- no component registry in use |

External dependency: Mapbox GL JS loaded via CDN (`<script>` tag). This is a widely-used, well-audited mapping library from Mapbox Inc. No registry vetting gate applies to CDN-loaded libraries (only to shadcn registries).

Note: Mapbox GL JS requires a free API access token. The token should be committed as a JS constant (not in .env) since this is a public static site with no server -- the token is inherently public. Mapbox free tier (50,000 map loads/month) is well within wedding website traffic expectations.
