# Phase 3 — UI Design Contract

> Visual and interaction contract for the Guest Graph phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS) |
| Icon library | none (CSS-drawn UI elements: search icon, filter toggles, close buttons) |
| Font | Playfair Display (heading) + Source Sans 3 (body) via Google Fonts CDN |
| Graph library | D3.js v7 via CDN (Claude's Discretion -- best support for force-directed layout, hierarchical tree layout, expand-in-place node manipulation, SVG-based rendering for photo circles and cluster regions, and CDN availability without build tools) |

**Source:** Phase 1 design system baseline, CONTEXT.md Claude's Discretion (library selection)

**D3.js rationale:** D3 is the strongest choice for this phase because it natively supports both force-directed (`d3-force`) and hierarchical tree (`d3-hierarchy`) layouts in a single library, provides fine-grained SVG control needed for photo-circle nodes and expand-in-place animations, handles the soft colored cluster regions via Voronoi or convex hull overlays, and is available as a single CDN script with no build step required. Cytoscape.js was considered but lacks the SVG flexibility needed for the rich node styling (photo circles with fallback initials). vis.js was considered but its tree layout support is weaker.

**Source:** Phase 1 UI-SPEC spacing scale, carried forward

### Cluster Region Colors

Each relationship group gets a soft, warm-toned fill behind its cluster (D-10). Colors are the existing palette at `--color-cluster-opacity` (8%):

| Group | Fill Color (at 8% opacity) | Notes |
|-------|---------------------------|-------|
| Arash's Family | `--color-journey-arash` (#2A9D8F at 8%) | Teal tint -- same family color as timeline |
| Natalie's Family | `--color-journey-natalie` (#D4A843 at 8%) | Gold tint -- same family color as timeline |
| College Friends | `--color-dusty-rose` (#C9928E at 8%) | Warm rose tint |
| Work (Meta) | `--color-terracotta` (#C2704F at 8%) | Terracotta tint |
| Other groups | Rotate through above 4 tint colors | Ensure adjacent clusters use different tints |

Cluster region shapes: convex hull with 24px padding and 20px border-radius corners (`--radius-lg`). Regions must feel "warm and organic, not like rigid bounding boxes" per CONTEXT.md specifics.

Accent reserved for:
- Active filter button highlight (selected group filter)
- Active view toggle (Social Graph / Family Tree)
- Search icon in the search bar
- Natalie and Arash's node border (couple emphasis)

Elements that must NOT use accent:
- Default node borders (use `--color-node-default`)
- Edge lines (use `--color-edge-default`)
- Cluster region fills (use group-specific tints at 8%)
- Node name label text

## Shape & Motion

### Inherited (from Phase 1)

| Property | Token | Value |
|----------|-------|-------|
| Border radius (small) | `--radius-sm` | 6px |
| Border radius (medium) | `--radius-md` | 12px |

| Shadow (soft) | `--shadow-soft` | 0 2px 8px rgba(61, 43, 31, 0.08) |
| Shadow (medium) | `--shadow-medium` | 0 4px 16px rgba(61, 43, 31, 0.12) |
| Transition (fast) | `--transition-fast` | 150ms ease |
| Transition (normal) | `--transition-normal` | 250ms ease |

### Phase 3 Additions

| Property | Value | Usage |
|----------|-------|-------|
| Node expand duration | 300ms ease-out | Expand-in-place animation when node is tapped (D-11) |
| Node collapse duration | 200ms ease-in | Collapse back to default size |
| Force simulation settle | ~3000ms (D3 default alpha decay) | Initial force layout settling after render |
| View switch crossfade | 400ms ease | Transition between social graph and family tree layouts |
| Dim non-connected | 200ms ease | Opacity fade on non-connected nodes when one is expanded |
| Search zoom-to | 500ms ease-in-out | Pan and zoom to searched node |
| Cluster region reveal | 200ms ease | Fade-in of cluster regions on initial load |

### Node Sizes

| Node Type | Circle Diameter (social graph) | Circle Diameter (family tree) |
|-----------|-------------------------------|-------------------------------|
| Natalie & Arash (couple) | 56px | 64px |
| Photo node (other guests) | 40px | 52px |
| Initials-only node | 36px | 48px |

Family tree nodes are intentionally larger than social graph nodes per D-16 ("nodes are bigger and always show name + photo").

### Node Circle Styling

- Circular clip path on profile photo, centered in node
- 2px `--color-node-default` border for regular nodes
- 3px `--color-node-couple` border for Natalie and Arash
- Nodes without photos: warm cream (#FFF8F0) fill with initials in 14px Source Sans 3 semibold, `--color-text-secondary` color
- Box shadow on hover: `--shadow-soft`

### Edge Styling (Claude's Discretion)

- Default edges: 1px stroke, `--color-edge-default` (warm brown at 15% opacity)
- Typed relationship edges (family, best friend, roommate, introduced-us): 1.5px stroke, same color but at 25% opacity
- No edge labels in default view (too cluttered at 80-150+ nodes)
- When a node is expanded, its connected edges become `--color-edge-highlight` (50% opacity) and connected node labels become bold
- Edge labels appear ONLY on highlighted edges when a node is expanded: 11px Source Sans 3 regular, `--color-text-secondary`, positioned at edge midpoint

### Connection Highlighting (Claude's Discretion)

When a node is expanded:
1. Connected edges transition to `--color-edge-highlight` (200ms)
2. Connected nodes remain at full opacity, their labels become semibold
3. Non-connected nodes dim to `--color-node-dim` opacity (0.25) over 200ms
4. Non-connected edges dim to 0.05 opacity over 200ms
5. Edge type labels appear on highlighted edges only
6. Clicking the background or pressing Escape collapses the node and restores full opacity

Usage rules:
- Search input: `--radius-sm` (6px)
- Filter/toggle buttons: `--radius-sm` (6px)
- Expanded node detail card (inline): `--radius-md` (12px)
- Cluster region corners: `--radius-lg` (20px) with convex hull path smoothing
- Node circles: 50% border-radius (perfect circle)

**Source:** CONTEXT.md D-09 (flat network, couple emphasis), D-10 (cluster regions), D-11 (expand in place), D-18 (photo circles), Claude's Discretion (edge styling, connection highlighting)

## Layout & Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile | < 768px | Controls stack vertically; graph is full-width with touch gestures; expanded node detail is a bottom sheet |
| Desktop | >= 768px | Controls in a single row; graph fills remaining viewport; expanded node detail is inline |

### Desktop Layout (>= 768px)

```
+-----------------------------------------------+
|  [Site Header: Logo + Nav]                     |
+-----------------------------------------------+
| [Search___________] [Everyone | Family Tree]   |
| [All] [Arash Fam] [Nat Fam] [College] [Meta]  |
+-----------------------------------------------+
|                                                |
|              GRAPH CANVAS (SVG)                |
|                                                |
|      (o)---(o)     (o)                         |
|       |     |      / \                         |
|      (o)   (O)---(O)  (o)                      |
|             N  &  A                            |
|      (o)---(o)     (o)---(o)                   |
|                                                |
+-----------------------------------------------+
```

- Header: shared site header with nav (standard, inherited)
- Controls row: search input (flex: 1, max-width 320px) + view toggle (right-aligned), gap `--space-md`
- Filter bar: below controls, horizontal scrollable row, gap `--space-sm` between pills
- Graph canvas: fills remaining viewport height (`calc(100vh - header - controls - filter - padding)`), minimum height 400px
- Graph canvas has `overflow: hidden` with D3 zoom/pan transform applied to inner `<g>` element

### Mobile Layout (< 768px)

```
+---------------------------+
| [Site Header]             |
+---------------------------+
| [Search________________]  |
| [Everyone | Family Tree]  |
| [All][Arash][Nat][Coll]>> |
+---------------------------+
|                           |
|     GRAPH CANVAS          |
|    (pinch to zoom)        |
|                           |
+---------------------------+
```

- Search: full width minus `--space-md` padding on each side
- View toggle: full width, centered
- Filter bar: horizontally scrollable with overflow, left-aligned, right fade hint (linear-gradient mask) to indicate scrollability
- Graph canvas: fills remaining viewport, touch-enabled with pinch-zoom and pan (D3 zoom behavior)
- Expanded node on mobile: slides up as a bottom sheet (positioned at bottom of viewport, max-height 40vh, `--color-warm-white` bg, `--shadow-medium`, `--radius-md` on top corners). This avoids the inline expansion obscuring the graph on small screens.

### View Toggle UI (Claude's Discretion)

Two-button segmented toggle (not tabs, not dropdown):
- "Everyone" = social graph view (force-directed)
- "Family Tree" = hierarchical tree view

Active button: `--color-terracotta` background, white text
Inactive button: `--color-warm-white` background, `--color-text-primary` text, 1px `--color-dusty-rose` border
Toggle container: `--radius-sm` with overflow hidden (buttons flush together)
Minimum width per button: 100px
Height: 36px (desktop), 40px (mobile)

### Family Tree Layout

```
+-----------------------------------------------+
| [Search___________] [Everyone | Family Tree]   |
| [Both Families] [Natalie's] [Arash's]          |
+-----------------------------------------------+
|                                                |
|   Natalie's Family    |    Arash's Family      |
|                       |                        |
|      Grandma          |         Grandpa        |

|     Mom   Uncle       |       Dad   Aunt       |
|      |                |        |               |
|   Natalie ============== Arash                 |

```

- Both families displayed side by side, connected at the couple node (D-14)
- Tree flows top-down (ancestors at top, couple at center/bottom)
- Filter buttons replace the group filter bar when in Family Tree view
- Horizontal scrollable on mobile if tree is wider than viewport
- Tree branch lines use family colors: teal for Arash's branches, gold for Natalie's branches
- Couple connector line: `--color-terracotta` (accent, the union point)

## Interaction States

### Search

| State | Visual Treatment |
|-------|-----------------|
| Empty | Placeholder text "Search for a guest..." in `--color-text-secondary` |
| Typing | Text in `--color-text-primary`, clear button (X) appears at right edge |
| Match found | Graph pans and zooms to matching node over 500ms, node pulses once (scale 1.2x then back to 1.0x over 300ms) |
| No match | Below search input, inline message: "No guests found for \"{query}\". Try a different name." in `--color-text-secondary`, 14px |
| Focused | 2px `--color-terracotta` border, `--shadow-soft` |

### Nodes (Social Graph)

| State | Visual Treatment |
|-------|-----------------|
| Default | Photo circle at 40px (or 56px for couple), name label below, `--color-node-default` 2px border |
| Hover (desktop) | `--shadow-soft` appears around circle, cursor: pointer |
| Expanded | Node grows to 80px (couple: 96px). Surrounding nodes push outward (D3 force repulsion increase). Detail card appears inline (desktop) or as bottom sheet (mobile). Connected edges highlight, non-connected nodes dim. |
| Dimmed | Opacity `--color-node-dim` (0.25), name label hidden |
| Search result | Zoom-to animation + single pulse (scale 1.2x -> 1.0x over 300ms) |
| Filtered out | Node removed from force simulation with fade-out (200ms). Remaining nodes re-settle. |

### Nodes (Family Tree)

| State | Visual Treatment |
|-------|-----------------|
| Default | Photo circle at 52px (couple: 64px), name label always visible below, no dimming behavior |
| Hover | `--shadow-soft` around circle |
| Expanded | Node grows to 88px. Detail card appears below the node inline. Tree layout does not rearrange (static positions). |

### View Toggle

| State | Visual Treatment |
|-------|-----------------|

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Language | `lang="en"` on `<html>` (inherited from Phase 1 template) |
| Graph aria label | `aria-label="Interactive guest relationship visualization"` on SVG container |
| Graph role | `role="img"` on SVG container with `aria-describedby` pointing to a visually-hidden text summary: "{N} guests shown with {M} connections" |
| Node buttons | Each node is a `<g>` with `role="button"`, `tabindex="0"`, `aria-label="{Guest Name}"` |
| Expanded node | `aria-expanded="true/false"` on node groups, expanded detail card has `role="dialog"` with `aria-label="{Guest Name} details"` |
| Search input | `<input type="search" aria-label="Search guests">`, `role="searchbox"` implicit |
| Search results | `aria-live="polite"` region announcing match count or "No guests found" |
| View toggle | `role="radiogroup"` with `aria-label="View mode"`, each button: `role="radio"`, `aria-checked="true/false"` |
| Filter buttons | `role="group"` with `aria-label="Filter by group"`, each button: `aria-pressed="true/false"` |
| Keyboard: graph | Tab through nodes (in DOM order), Enter/Space to expand, Escape to collapse |
| Keyboard: search | Tab to search input, type to filter, Enter to zoom to first match |
| Keyboard: filters | Tab through filter buttons, Enter/Space to toggle |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` -- disable force simulation animation (render final positions instantly), disable expand/collapse transitions (instant), disable zoom-to animation (instant), disable crossfade between views (instant swap) |
| Touch targets | All interactive elements minimum 44x44px (nodes via invisible hit area, search input height, filter buttons, view toggle buttons) |
| Color contrast | Node name labels (#3D2B1F) on cream (#FFF8F0): 10.5:1 ratio (exceeds AAA). Terracotta (#C2704F) on white: 4.6:1 (exceeds AA for large text). All text meets WCAG AA minimum. |
| Focus visible | All interactive elements: 2px `--color-terracotta` outline with 2px offset |
| Skip to content | Visually-hidden "Skip to guest graph" link before the controls for keyboard/screen reader users |
| Mobile bottom sheet | Focus trapped within bottom sheet when open, Escape closes, focus returns to triggering node |

**Source:** Phase 1 and Phase 2 UI-SPEC accessibility requirements, extended for interactive graph concerns
