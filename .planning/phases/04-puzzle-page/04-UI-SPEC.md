## Spacing Scale

Inherited from Phase 1 design system (`narsh2026/styles.css`). No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Card inner padding between emoji and name, color tag dot inset |
| sm | 8px | Grid gap (desktop), card inner padding, button icon-text gaps |
| md | 16px | Page content padding, space between game controls and grid, modal inner padding |
| lg | 24px | Section spacing (between puzzle game and daily links), modal outer margin |
| xl | 48px | Top padding below header (page-level spacing) |
| 2xl | 64px | Not used in this phase |

Exceptions:
- Card tap targets: entire card surface is tappable, minimum card size 72px x 92px (ensures 44px minimum touch target is exceeded)
- Game control buttons (Hint, Inspect, Share): minimum 44px height, 12px horizontal padding
- Modal close button: 44x44px touch target

**Source:** Phase 1 UI-SPEC spacing scale, carried forward

## Typography

Inherited from Phase 1 design system. Two weights only: 400 (regular) and 700 (bold).

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Heading (page h1) | 28px | 700 (bold) | 1.2 | Playfair Display |
| Section heading (h2), modal heading | 20px | 700 (bold) | 1.2 | Playfair Display |
| Body, modal body, daily puzzle title | 16px | 400 (regular) | 1.6 | Source Sans 3 |
| Small / Label | 12px | 400 (regular) | 1.4 | Source Sans 3 |

Mobile overrides (below 768px):
- Heading (h1): 24px

Phase 4 specific (all sizes from the 4-size scale above):
- Card emoji (profession icon): 28px line-height 1.0 (centered in card face)
- Card character name: 12px Source Sans 3 bold (700), line-height 1.2, max 2 lines with ellipsis overflow
- Card profession label: 12px Source Sans 3 regular (400), line-height 1.2, `--color-text-secondary`
- Clue text (flipped card back): 12px Source Sans 3 regular (400), line-height 1.4
- Criminal/innocent role label: 12px Source Sans 3 bold (700), uppercase
- Game stats bar (timer, mistakes, hints): 12px Source Sans 3 bold (700)
- Modal heading: 20px Playfair Display bold (700), line-height 1.2
- Modal body text: 16px Source Sans 3 regular (400), line-height 1.5
- Button labels (Hint, Inspect, Share): 12px Source Sans 3 bold (700)
- Daily puzzle link card title: 16px Source Sans 3 bold (700)
- Daily puzzle link card description: 12px Source Sans 3 regular (400), `--color-text-secondary`

Font loading: No changes to Source Sans 3 import needed (existing pages load 400; 700 is already loaded for Playfair Display headings). The puzzle page's Google Fonts link remains:
```
Source+Sans+3:wght@400
```

**Source:** Phase 1 UI-SPEC typography, consolidated to 4 sizes (12/16/20/28) and 2 weights (400/700) for game-specific text roles, CONTEXT.md D-04 (emoji faces + name + profession)

### Color Tag Palette (Game Note-Taking)

Per CONTEXT.md D-05, players can mark cards with colored tags for note-taking. These are functional game markers, not decorative -- they must be maximally distinguishable from each other and from card states.

| Index | Color | Hex | CSS Class |
|-------|-------|-----|-----------|
| 0 | Transparent (no tag) | transparent | `.tag-none` |
| 1 | Red | #E74C3C | `.tag-red` |
| 2 | Orange | #E67E22 | `.tag-orange` |
| 3 | Yellow | #F1C40F | `.tag-yellow` |
| 4 | Green | #27AE60 | `.tag-green` |
| 5 | Blue | #3498DB | `.tag-blue` |
| 6 | Purple | #8E44AD | `.tag-purple` |

Tag rendering: 8px diameter dot positioned at top-right corner of card, 4px inset from edges. Only visible when tag index > 0.

Accent reserved for:
- Primary action buttons (Show Hint, Share Results)
- Criminal card reveal background
- Game complete celebration heading
- Correct-guess speech bubble left border

Elements that must NOT use accent:
- Card front surfaces (use `--color-cream`)
- Innocent reveal (use teal `--color-innocent`)
- Color tag dots (use their own distinct palette)
- Daily puzzle link cards (use `--color-warm-white`)

**Source:** Phase 1 UI-SPEC color contract, Phase 2 journey colors (teal reused for innocent), CONTEXT.md D-05 (color tags), D-12 (warm wedding aesthetic), D-13 (card styling adapts Clues by Sam)

## Shape & Motion

### Inherited (from Phase 1)

| Property | Token | Value |
|----------|-------|-------|
| Border radius (small) | `--radius-sm` | 6px |
| Border radius (medium) | `--radius-md` | 12px |

| Transition (normal) | `--transition-normal` | 250ms ease |

### Phase 4 Additions

| Property | Value | Usage |
|----------|-------|-------|
| Card flip duration | 500ms ease | CSS 3D `rotateY(180deg)` transition on `.puzzle-card-inner` |
| Card select pulse | 200ms ease | Brief scale(1.03) pulse when card is tapped before guess dialog |
| Incorrect shake | 400ms ease | `translateX(-4px, 4px, -4px, 4px, 0)` keyframe shake on wrong guess |
| Incorrect flash | 300ms ease | Background flash to `--color-incorrect-flash` then back |
| Dim toggle | 200ms ease | Opacity transition for inspect/dim mode |
| Modal enter | 250ms ease-out | Scale(0.95) + opacity(0) to scale(1) + opacity(1) |
| Modal exit | 150ms ease-in | Reverse of enter |
| Color tag cycle | instant | No transition -- immediate color swap on long-press/right-click |
| Hint highlight pulse | 600ms ease infinite | Subtle box-shadow pulse on hint-highlighted cards (alternating `--shadow-soft` to `--shadow-medium`) |
| Share copied feedback | 2000ms | "Copied!" text visible for 2 seconds then fades |

### Card Dimensions

| Viewport | Card Width | Card Height | Grid Gap |
|----------|-----------|-------------|----------|
| Desktop (>= 768px) | ~120px (1fr of 4-col grid, max-width 528px grid) | ~150px (aspect-ratio 4:5) | 8px |
| Mobile (480-767px) | ~90px (1fr) | ~112px (aspect-ratio 4:5) | 8px |
| Small mobile (< 480px) | ~76px (1fr) | ~95px (aspect-ratio 4:5) | 4px |

Card aspect ratio is 4:5 (width:height). The grid has a `max-width: 528px` on desktop to prevent cards from becoming overly large. Cards have `--radius-md` (12px) corners.

### Card Structure

```
+------------------+
| [tag dot]        |  <- 8px dot, top-right, 4px inset
|                  |
|      [emoji]     |  <- 28px, centered
|                  |
|   [name text]    |  <- 12px bold, centered, max 2 lines
|  [profession]    |  <- 12px regular, centered, 1 line
+------------------+
```

Card front: `--color-cream` background, 1px `--color-dusty-rose` border, `--radius-md` corners, `--shadow-soft`.

Card back (flipped, criminal):
```
+------------------+
| CRIMINAL         |  <- 12px bold, --color-criminal, uppercase
|                  |
| "Clue text here  |  <- 12px regular, --color-text-primary
|  referencing     |
|  other chars"    |
|                  |
+------------------+
```
Background: `--color-criminal-light` tint. Left border: 3px solid `--color-criminal`.

Card back (flipped, innocent): Same layout. Background: `--color-innocent-light` tint. Left border: 3px solid `--color-innocent`. Label reads "INNOCENT" in `--color-innocent`.

### CSS 3D Flip

Card flip uses `perspective: 800px` on `.puzzle-card`, `transform-style: preserve-3d` on `.puzzle-card-inner`, `backface-visibility: hidden` (with `-webkit-` prefix) on both `.puzzle-card-front` and `.puzzle-card-back`. The back face has `transform: rotateY(180deg)` as its default state.

Usage rules:
- Puzzle cards: `--radius-md` (12px)
- Game control buttons: `--radius-sm` (6px)
- Modals: `--radius-md` (12px)
- Daily puzzle link cards: `--radius-md` (12px)
- Speech bubble (correct guess): `--radius-sm` (6px)

**Source:** RESEARCH.md Pattern 4 (Card Flip CSS), Pitfall 5 (Safari backface), CONTEXT.md D-04 (card front/back states), D-13 (card styling)

## Interaction States

### Puzzle Cards

| State | Visual Treatment |
|-------|-----------------|
| Default (unflipped) | Card front visible: emoji, name, profession. `--color-cream` bg, 1px `--color-dusty-rose` border, `--shadow-soft`. Cursor: pointer. |
| Hover (desktop) | Elevation increases: `--shadow-medium`. Slight scale: `scale(1.02)`. Transition: `--transition-fast`. |
| Tapped/clicked | Guess dialog appears adjacent to card. Card gets 2px `--color-card-selected` (golden) ring. |
| Correct guess (criminal) | Card flips (500ms). Back face shows "CRIMINAL" label in `--color-criminal`, clue text, `--color-criminal-light` background tint, 3px `--color-criminal` left border. Speech bubble with clue text appears briefly (3s). |
| Correct guess (innocent) | Card flips (500ms). Back face shows "INNOCENT" label in `--color-innocent`, clue text, `--color-innocent-light` background tint, 3px `--color-innocent` left border. Speech bubble with clue text appears briefly (3s). |
| Incorrect guess | Card shakes (400ms keyframe animation). Brief `--color-incorrect-flash` background flash (300ms). Mistakes counter increments. Card stays unflipped. |
| Not deducible | Wrong guess modal appears. Card returns to default state. No mistakes increment. |
| Already flipped | No interaction. Cursor: default. Card shows back face permanently. |
| Dimmed (inspect mode on, card already read) | Opacity: `--color-card-dimmed` (0.35). Transition: 200ms ease. Still clickable to un-dim (toggle). |
| Hint highlighted | Subtle `--shadow-medium` pulse animation (600ms infinite). Draws attention to suggested next flip. Stops when hint is hidden or card is flipped. |
| Color tagged | 8px colored dot at top-right corner with 4px inset. Color from tag palette. |

### Game Controls

| Control | Default State | Active/Toggle State | Disabled State |
|---------|---------------|---------------------|----------------|
| Show Hint button | Outlined: 1px `--color-terracotta` border, `--color-terracotta` text, transparent bg | Active: `--color-terracotta` bg, white text. Label changes to "Show More" or "Hide Hint" | Opacity 0.5, cursor: not-allowed (when all hints exhausted) |
| Inspect button | Same outlined style as Hint | Toggled on: `--color-terracotta` bg, white text, label "Inspecting" | N/A |
| Share Results button | `--color-terracotta` bg, white text (always filled -- primary action feel) | "Copied!" text for 2s, then reverts | N/A |

### Color Tagging

| Interaction | Trigger |
|-------------|---------|
| Desktop | Right-click on card opens cycle (suppress native context menu with `preventDefault`) |
| Mobile | Long-press (300ms `touchstart` hold, cancel on `touchmove` or `touchend` before 300ms) |
| Behavior | Each trigger cycles to next color in the 7-item palette (transparent -> red -> orange -> yellow -> green -> blue -> purple -> transparent). No picker UI -- single action cycles through. |

### Daily Puzzle Link Cards

| State | Visual Treatment |
|-------|-----------------|
| Default | `--color-warm-white` background, `--shadow-soft`, `--radius-md` corners. Emoji at left (32px), title + description stacked at right. |
| Hover (desktop) | Lift: `translateY(-2px)`, shadow: `--shadow-medium`. Transition: `--transition-fast`. |
| Active/pressed | `translateY(0)`, shadow: `--shadow-soft` (pressed down feel). |

## Mobile Touch Interactions

| Gesture | Action |
|---------|--------|
| Single tap (unflipped card) | Show guess dialog adjacent to card |
| Single tap (Criminal/Innocent button) | Submit guess |
| Single tap (flipped card) | No action (card is resolved) |
| Single tap (background / outside dialog) | Dismiss guess dialog |
| Long-press (300ms) on card | Cycle color tag |
| Tap game control button | Trigger hint / toggle inspect / share results |
| Tap daily puzzle card | Navigate to external puzzle (new tab) |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Language | `lang="en"` on `<html>` (inherited from Phase 1 template) |
| Game section aria | `aria-label="Clues puzzle game"` on `section.puzzle-game` |
| Card grid role | `role="grid"` on `.puzzle-grid`, each row has `role="row"` (implicit from grid), each card has `role="gridcell"` |
| Card buttons | Each unflipped card: `role="button"`, `tabindex="0"`, `aria-label="{Name}, {profession}"` |
| Card flipped state | Flipped cards: `aria-label="{Name}, {role} -- {clue text}"`, `aria-disabled="true"` |
| Guess dialog | `role="dialog"`, `aria-label="Guess role for {Name}"`, focus trapped within dialog |
| Modals | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal heading. Focus trapped. Escape closes. |
| Stats bar | `aria-live="polite"` on mistakes and hints counters (announces changes) |
| Timer | `aria-label="Elapsed time: {formatted time}"` updated every 10 seconds (not every second, to avoid screen reader noise) |
| Hint button | `aria-pressed="true/false"` for toggle state |
| Inspect button | `aria-pressed="true/false"` for toggle state |
| Daily puzzle links | Standard `<a>` elements with descriptive text. `target="_blank"` with `rel="noopener noreferrer"`. Include `aria-label="{Name} -- opens in new tab"`. |
| Keyboard: cards | Tab through cards in grid order (left-to-right, top-to-bottom). Enter/Space opens guess dialog. |
| Keyboard: guess dialog | Tab between Criminal and Innocent buttons. Enter/Space to confirm. Escape to cancel. |
| Keyboard: controls | Tab to Hint, Inspect, Share buttons. Enter/Space to activate. |
| Keyboard: modals | Focus trapped in modal. Tab through focusable elements. Escape closes. |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` -- disable card flip animation (instant class toggle), disable shake animation, disable hover scale, disable hint pulse, disable daily card lift. All state changes still occur, just without animation. |
| Touch targets | All interactive elements minimum 44x44px (cards exceed this, buttons enforce min-height 44px) |
| Color contrast | Text on cream (#3D2B1F on #FFF8F0): 10.5:1 (AAA). Criminal label (#C2704F) on criminal-light bg: test and ensure AA. Innocent label (#2A9D8F) on innocent-light bg: test and ensure AA. White text on terracotta buttons: 4.6:1 (AA large text). |
| Focus visible | All focusable elements: 2px `--color-terracotta` outline, 2px offset |
| `textContent` only | All dynamic text rendering must use `textContent` or `createTextNode`, never `innerHTML`, to prevent XSS from puzzle data (per RESEARCH.md security domain) |

**Source:** Phase 1 and Phase 3 UI-SPEC accessibility patterns, RESEARCH.md security domain (textContent), Pitfall 4 (touch targets)
